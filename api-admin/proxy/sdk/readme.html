超级签名安装助手接入手册

准备工作

如果需要接入超级签名安装助手，您需要准备以下信息

  • 短链接安装地址

集成步骤

  1. 添加 meta 标签

    在需要集成安装助手的页面添加如下代码。

<meta name="vip-short-url" content="{短链接安装地址}" />
<meta name="vip-auto" content="1" />

  1. 添加助手

    在页面底部添加如下代码

<script type="text/javascript" src="https://zc.pgyer.com/static/sdk/vip.min.js"></script>

  1. 绑定安装按钮

    在安装按钮对应的 HTML 元素上添加如下属性

role="vip.install"

例如

<button role="vip.install">安装</button>

meta 标签参数说明

安装助手依靠 meta 标签来标记数据, 不同的 name 值表示的含义如下表

name 含义
vip-short-url 安装短链接地址(必填)
vip-auto 0 表示不自动安装, 1 表示自动安装(推荐)

回调函数

signSDK.onLoad

当助手成功加载时, 会回调注册在 signSDK.onLoad 方法的函数, 并传入一个 env 对象。示例代码如下:

signSDK.onLoad = function (env) {
  console.log(env);
}

evn 对象包含以下属性

属性名 含义
iOS boolean 设备是否为 iOS 操作系统
safari boolean 设备是否在使用 safari 浏览器
version string 当前 iOS 系统版本 (非 iOS 系统返回 null)

signSDK.onPhaseChange

在安装过程中, 每进入不同安装阶段, 或阶段进度发生变化时回调用注册在 signSDK.onPhaseChange 方法的函数, 并传入一个 phase 对象。示例代码如下:

属性名 含义
phase int 当前阶段: -1:安装错误, 0:初始化, 1:获取设备信息, 2:验证设备信息, 3:准备下载安装
progress int 估计(预测)的进度值(不推荐使用)

例如:

signSDK.onPhaseChange = function (phase) {
    if (phase.phase === 0) {
        // 正在准备
    } else if (phase.phase === 1) {
        // 安装描述文件以完成验证
    } else if (phase.phase === 2) {
        // 正在验证设备
    } else if (phase.phase === 3) {
        // 打开后点击安装即可安装
    } else if (phase.phase === -1) {
        // 安装失败, 重新安装
    }
}

控制台报错

消息 内容
This Browser May Not Fit For This SDK! 助手不能在此浏览器下工作
Missing Meta Data: vip-short-url! 缺少 meta 数据 (详见集成过程中的第一步)
Invalid Meta Data: vip-short-url! vip-short-url 数据不合法
Install Button Not Found! 安装按钮没有绑定
